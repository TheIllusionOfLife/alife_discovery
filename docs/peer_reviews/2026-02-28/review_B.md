# ALife Conference Full Paper 査読

## 総評
本稿は、目的関数なしのブロック世界において構造的に非自明なエンティティが出現するかを、Assembly Theory（AT）を用いて検証した大規模実験報告である。7M超の観測とルール表のランダムサンプリングというスケール感は評価でき、負の結果を明確に提示する点もALife領域として意義がある。一方で、用いている assembly index の定義と計算法がATの意図する「構造特異性」を十分に捉えているかが不明確であり、また評価の独立性・統計的妥当性・パラメータ探索の幅に不足がある。フルペーパーとしては、測度の妥当性と境界条件の一般性をもう一段補強してほしい。

## 強み
- 目的関数なしのALifeにおける「負の結果」を大規模に提示し、研究コミュニティに有用なベースラインを提供している。
- 実験規模（5,000シミュレーション、7M観測、1,000ルール表）と再現性のあるパイプラインが明確に記述されている。
- 結果のまとめ方が簡潔で、図表（サイズ分布、監査、ギャラリー）が研究の主張を支えている。

## 主な懸念・弱み
- **assembly index の定義・計算法の妥当性が曖昧**：本文の定義は「エッジ追加回数の最小値」に読めるが、それは多くのグラフで |E| と同値になり、構造差を反映しにくい。DPが実際に算出している量（最大深さ最小化）がATの assembly index と一致するのか明示が必要。
- **「サイズのみで説明される」という結論の根拠が弱い**：ヌルモデルは次数列を保存しており、純粋にサイズのみの比較ではない。サイズ・密度・次数列のどの要因が支配的かの分析が不足している。
- **統計手続きの弱さ**：n_shuffle=20は分布推定として少なく、正規性仮定で μ+2σ を用いた有意判定も妥当性が不明。非パラメトリックな p 値評価などが望ましい。
- **パラメータ探索の狭さ**：密度・格子サイズ・ブロックタイプ数・近傍半径などの系統的スイープがなく、「境界条件」の一般性を主張するには弱い。

## 詳細コメント（改善提案）
- assembly index の定義を式で明示し、同一サイズ・同一エッジ数の異なるグラフで値が異なる例を示してほしい。異ならない場合は、測度自体が構造特異性を捉えていない可能性がある。
- DPの「分解可能条件（共有エッジを持つ2連結成分）」の根拠と、ATのサブオブジェクト再利用との関係を整理してほしい。
- ヌルモデルの double-edge swap の回数・実際のランダム化程度を明記し、小規模グラフでの有効性を検証してほしい。
- 7M観測の多くが同一エンティティの時間的反復である可能性が高い。独立サンプルとして扱うのか、持続時間や一意タイプ単位での評価なのかを明確化してほしい。
- 「サイズによる説明」を主張するなら、サイズ・次数列・局所クラスタリング等のグラフ統計量での回帰や分散分解を示すと説得力が増す。
- 追加の軽量スイープ（密度・格子サイズ・ブロックタイプ数）を行い、「境界条件」の範囲を広げられるとフルペーパーとして強くなる。

## 著者への質問
1. assembly index として実装したDPの出力は、ATの標準定義と形式的に一致しますか？一致する場合、対応関係を本文で明示できますか？
2. ヌルモデルの double-edge swap は各グラフにつき何回行っていますか？小規模グラフで十分に混合している根拠はありますか？
3. 7M観測の「独立性」はどのように扱っていますか？タイプ単位とインスタンス単位の両方の分析を示せますか？
